**Live site:** [https://banana-countdown.vercel.app/](https://banana-countdown.vercel.app/)

## Overview

Banana Countdown is an end-to-end Computer Vision web app with a two-stage pipeline:

| Stage | Model | Output |
|---|---|---|
| 1 — Detection & Classification | YOLOv11 (fine-tuned) | Bounding box + **ripeness label** + confidence score |
| 2 — Shelf-life Regression | ResNet-18 (fine-tuned) | **Days remaining** until spoilage (continuous) |

Upload a photo → get an annotated image showing, for each detected banana: its ripeness category (from YOLO), confidence score (from YOLO), and days-remaining countdown (from CNN).

> **Important:** Ripeness classification (Fresh Ripe, Overripe, etc.) is produced **exclusively by YOLO** in Stage 1. The ResNet-18 CNN predicts only the numeric shelf-life value and does not re-classify ripeness.

---

## Ripeness Classes (6)

| Class | Description |
|---|---|
| Fresh Unripe | Newly harvested, fully green |
| Unripe | Green, not yet ready |
| Fresh Ripe | Just turned yellow |
| Ripe | Perfect eating condition |
| Overripe | Soft, browning — best for baking |
| Rotten | Discard |

## Environment Setup

Ensuring your environment is setup correctly is an important step to running the application locally.

```bash
cp .env.example .env
# Fill in your credentials in .env
```

`.env.example`:
```bash
KAGGLE_USERNAME=your_kaggle_username
KAGGLE_KEY=your_kaggle_api_key
WANDB_API_KEY=your_wandb_api_key
ROBOFLOW_API_KEY=your_roboflow_api_key
```

Backend YOLO inference thresholds (in `.env`):
```bash
YOLO_CONF=0.5
YOLO_IOU=0.45
YOLO_MAX_DET=10
YOLO_IMGSZ=640
YOLO_AGNOSTIC_NMS=false
YOLO_AUGMENT=false
```

## Quickstart

### 1. Clone & install
```bash
git clone https://github.com/YOUR_USERNAME/banana-countdown.git
cd banana-countdown
pip install -r backend/requirements.txt
```

### 2a. Download YOLO dataset (Roboflow, 7,546 images)
```bash
python training/download_yolo_data.py
# Downloads annotated banana ripeness data into data/yolo/
```

### 2b. Download regression dataset (Kaggle, 471 images)
```bash
kaggle datasets download -d anishkumar00/days-death-to-a-banana
unzip days-death-to-a-banana.zip -d data/regression_raw
```

### 3. Prepare regression dataset
```bash
python training/prepare_regression_data.py
# Parses filenames (banana_XX_day_YY.jpg), computes days_remaining = max_day - day_shot,
# and copies images into data/regression/train|val|test/<days>/ using 80/10/10 split.
```

### 4. Train models
```bash
# Stage 1 — YOLOv11 fine-tuning (detection + ripeness classification)
python training/train_yolo.py

# Stage 2 — ResNet-18 fine-tuning (shelf-life regression only)
python training/train_regression.py

# Windows GPU note: if training is slow, install CUDA PyTorch first:
# pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121
```

Both scripts log metrics to W&B under team `ENSF-617-group-16`, project `banana-countdown`.
Trained weights are automatically copied to `backend/models/`.

### 4b. Alternatively, use the pretrained models
```bash
wandb artifact get ENSF-617-group-16/banana-countdown/regression-best:latest \
  --root backend/models/

wandb artifact get ENSF-617-group-16/banana-countdown/yolo-best:latest --root backend/models
ls -R backend/models
```

### 5. Run the backend
```bash
cd backend
python app.py
# API at http://localhost:5000
```

### 6. Run the frontend
```bash
cd frontend
npm install
npm run dev
# UI at http://localhost:3000
```

## API

### `POST /predict`

**Request:** `multipart/form-data` with field `image` (JPG or PNG)

**Response:**
```json
{
  "detections": [
    {
      "class_label":    "Overripe",
      "confidence":     0.91,
      "bounding_box":   [120, 45, 380, 290],
      "days_remaining": 1.5
    }
  ],
  "annotated_image": "<base64-encoded JPEG>"
}
```

- `class_label` and `confidence` come from **YOLO (Stage 1)**
- `days_remaining` comes from the **ResNet-18 CNN (Stage 2)**
